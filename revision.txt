1) System Purpose and Business Flow
-----------------------------------
This project is an IT support/ticketing platform where users can:
1. Create and manage support tickets.
2. Assign tickets to teams and members.
3. Track ticket status and analytics.
4. Use AI to predict priority and support decision making.

Core flow:
1. User creates ticket from frontend form.
2. Backend receives request and runs priority prediction.
3. Backend saves ticket in MongoDB.
4. Socket events notify connected clients.
5. UI updates dashboards, lists, and detail pages.

5) Exact Tech Stack and Why It Matters
--------------------------------------
This section explains each major technology in plain terms:
1. What it is.
2. Where you used it.
3. What impact it had on your project.

Backend Runtime and API Layer
1. Node.js
- What it is:
  JavaScript runtime outside the browser.
- Where used:
  Entire backend runtime in backend/server.js and all backend JS files.
- Impact on your project:
  Lets you run backend logic in the same language family as frontend (JS/React), reducing context switching and speeding development.

2. Express.js
- What it is:
  Lightweight web framework for Node.js.
- Where used:
  Route/middleware server composition in backend/server.js and backend/routes/*.
- Impact on your project:
  Gives a clear API structure (auth, tickets, teams, users, ai, upload) and makes middleware-based security and validation easy to apply consistently.

3. MongoDB
- What it is:
  NoSQL document database.
- Where used:
  Primary data store for users, teams, tickets, comments.

6) API and Security Design
--------------------------
Backend middleware stack:
1. helmet for secure headers.
2. cors allowlist with credentials.
3. compression for payload reduction.
4. body parser with request size limits.
5. express-rate-limit global request throttling.

Security patterns:
1. JWT route protection via auth middleware.
2. Validation layer via express-validator.
3. Centralized error handling for safe responses.
4. Auth-required AI model health endpoint.


7) AI + ML Integration Architecture
-----------------------------------
Priority prediction is integrated in ticket create flow:
1. Frontend sends ticket payload.
2. Backend ticket controller calls predictPriority(title, description, customerTier).
3. mlPredict loads ONNX model/session if available.
4. Prediction output enriches stored ticket fields.
5. If model unavailable/fails, fallback heuristic ensures continuity.

The system therefore supports both:
1. Intelligent prediction path (ONNX).
2. Operational fallback path (rule-based scoring).


8) Real-Time Events
-------------------

1. ticket_created
2. ticket_updated
3. team-scoped variants using room names team_<teamId>
